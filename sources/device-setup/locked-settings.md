# Locked Settings

## Purpose
Locked Settings is the protected setup layer for configuring a shared iPad or approved iPhone deployment within the Applied Intelligence app.

## Goal
Keep the day-to-day user experience simple while allowing authorized roles to configure the device once for its intended use.

## Primary Device Direction
- iPad first
- iPhone acceptable

## Setup Principle
The device should be configured once by an authorized person, then kept simple for the role using it.

## Why This Exists
Floor users should not have to navigate management settings to solve floor problems.

## Locked Settings Should Control
- Device Name
- Assigned Department
- Assigned Role
- Enabled Modules
- Can Acknowledge
- Can Resolve
- Can Close
- Can Escalate
- Can View Management Dashboard
- Can Access Formal Reporting
- Can Export Spreadsheet Data

## Example Device Profiles
### Welding Booth iPad
- Role: Lead
- Department: Welding
- Focus: AI-Trace + AI-Connect handoff

### Supervisor iPad
- Role: Supervisor
- Department: Production
- Focus: AI-Trace + AI-Connect response and closure

### Office / Management Device
- Role: General Manager or Executive Management
- Department: Office / Multi-Department
- Focus: Dashboard, escalation, oversight, exports, setup

## Access Rule
Locked Settings should only be available to authorized management-level roles.

## Current Recommendation
Allow access to Locked Settings for:
- General Manager
- Executive Management

Optional future expansion can decide whether limited supervisor setup access is allowed.

## Boundary Rule
Role permissions should control what the user can do, but should not change the identity of the underlying agent boundaries.
